# 방문 지도용 CSV 변환기 (xlsx → CSV)

복지/방문 관리용 엑셀 명단을 **구글 마이맵(Google My Maps)에 바로 넣을 수 있는 CSV**로 바꿔주는
**완전 클라이언트사이드** 웹 도구. 서버 없음, 업로드 없음 — 모든 처리는 사용자 브라우저 안에서만 일어난다.

## 왜 이렇게 만드나 (배경)

- 원본 엑셀은 복지 대상자 명단이라 **주민번호·건강정보 등 고민감 개인정보**가 붙어있다.
- 이 데이터가 절대 외부 서버로 나가면 안 된다 → **JS로만 처리**, 네트워크 전송 0.
- 결과물은 정적 파일 하나라 **AWS S3 static hosting**에 그냥 올려두면 끝.
- 실사용자(비개발자)가 브라우저에서 엑셀 올리고 → 열 몇 개 고르고 → CSV 받는 흐름.

## 핵심 요구사항

### 기술 스택
- **단일 `index.html` 파일**로 완결 (self-contained). CSS/JS 인라인 또는 같은 폴더 소수 파일.
- 빌드 스텝 없이 브라우저에서 바로 열려야 함 (S3에 그대로 업로드 가능).
- 엑셀 파싱: **SheetJS(xlsx)** CDN 사용. `.xlsx`, `.xls` 둘 다 지원.
- **백엔드/API 호출 절대 금지.** fetch·XHR로 데이터 전송하는 코드가 있으면 안 됨.

### 기능 흐름 (3단계 UI)

**1단계 — 파일 올리기**
- 드래그앤드롭 + 클릭 선택 둘 다 지원.
- `.xlsx`/`.xls`만 허용. 아니면 안내 메시지.
- 파일은 `FileReader`로 로컬에서만 읽는다.

**2단계 — 열 매핑**
- 원본 엑셀이 지저분함: **맨 위에 제목 행이 있을 수 있고, 병합 셀 있고, 컬럼이 30개 넘음**(AA·AB·AC까지).
- 그래서 컬럼 위치를 하드코딩하지 말고 **사용자가 드롭다운으로 고르게** 한다.
- **헤더 행 자동 감지**: 상위 ~15행을 훑어서 `"주소"`가 든 셀이 있는 행을 헤더로 추정. 사용자가 수동으로 바꿀 수 있게 헤더 행 선택 UI도 제공(각 행의 앞 몇 개 셀 미리보기 표시).
- 헤더 행이 정해지면 그 행의 셀들로 드롭다운 3개를 채운다: **관리번호 열 / 이름 열 / 주소 열**.
- **자동 추천(auto-guess)** 로직:
  - 주소 → 헤더에 `"주소"` 포함하는 열
  - 이름 → `"세대주"`, `"성명"`, `"이름"`, `"성함"`, `"대상자"` 중 하나 포함 (이 엑셀은 이름이 **세대주** 열에 있음)
  - 관리번호 → `"관리번호"`, `"가구번호"`, `"등록번호"`, `"번호"`, `"No"` 중 하나 포함
- **개인정보 옵션 체크박스: "이름을 성씨만 남기기"** — 켜면 이름을 첫 글자만 남김(`김미자` → `김`). 기본 켜짐 권장.

**3단계 — 미리보기 & 내려받기**
- 출력 결과 **상위 5행을 표로 미리보기** + 전체 유효 건수 표시.
- `[CSV 내려받기]` 버튼 → Blob으로 다운로드.
- 주소 열이 선택 안 됐거나 유효 행이 0이면 다운로드 비활성화 + 이유 안내.

### 출력 CSV 스펙 (중요 — 마이맵 호환)
- 컬럼: `관리번호, 이름, 주소` (성씨만 옵션 켜면 헤더를 `관리번호, 성씨, 주소`로).
- **인코딩: UTF-8 + BOM** (`\uFEFF` 프리픽스). 안 그러면 엑셀/마이맵에서 한글 깨짐.
- CSV 이스케이프: 쉼표·따옴표·줄바꿈 든 필드는 큰따옴표로 감싸고 내부 따옴표는 `""`로 이스케이프.
- **주소가 빈 행은 제외.**
- 참고: 좌표(위도·경도) 열은 이 도구에서 안 만든다. 마이맵이 주소를 직접 해석하는 "주소 버전" 흐름용. (좌표 변환은 별도 단계 — 아래 참고)

### 예외 처리 (사과 말고 방향 제시)
- 파일 형식 틀림 → "xlsx 또는 xls 파일을 올려주세요."
- 주소 열 자동 감지 실패 → 드롭다운은 뜨되, 미선택 시 "주소 열을 선택하세요."
- 유효 행 0 → "주소가 있는 행이 없어요. 헤더 행이나 주소 열을 다시 확인하세요."

### 품질 기준
- 반응형(모바일까지), 키보드 포커스 보임, `prefers-reduced-motion` 존중.
- UI 문구는 **한국어**, 존댓말, 간결하게.

## 마이맵 import 흐름 (사용자 안내용 — UI에 짧게 넣으면 좋음)
1. PC 브라우저에서 https://www.google.com/maps/d → **새 지도 만들기**
2. 왼쪽 레이어 박스 안 **[가져오기]** 클릭 → 이 도구가 만든 CSV 업로드
3. 위치 열 = **주소**, 마커 제목 열 = **관리번호**(또는 이름) 선택
4. 폰에서 보기: 구글맵 앱 → **저장됨 → 지도**

## 향후 확장 (지금은 범위 밖, 하지 말 것)
- 한국 주소는 구글이 가끔 부정확하게 찍음. 정확도 필요하면 **좌표(위도·경도) 열**을 미리 붙여야 함.
  - 국내 지오코딩: 지오코더(gimi9), 통계청 SGIS, 또는 카카오 지오코딩 API.
  - 좌표까지 붙일 거면 출력 컬럼에 `위도, 경도` 추가하고, 마이맵 import 때 위치 열로 위도·경도 두 개 선택.
- 카카오 지오코딩 API를 클라이언트에서 부르려면 JS 키의 도메인 제한 설정 필요(키 노출 주의). → 이건 완전 로컬 처리가 아니게 되므로 개인정보 트레이드오프 있음. 별도 결정.

## 샘플 파일
- `sample_명단.xlsx` — **입력용** 테스트 엑셀. 실제 컬럼 구조(No·가구번호·세대주·주민번호·주소 등 29열)와
  지저분한 레이아웃(제목 행, 빈 행, 병합 흉내)을 재현한 더미 데이터. 개인정보는 모두 가짜값.
- 마이맵 import 형식 **출력** 샘플 CSV 2종:
  - `mymap_주소버전.csv` — `관리번호, 성씨, 주소` (이 도구의 기본 출력 형식)
  - `mymap_좌표버전.csv` — `관리번호, 성씨, 주소, 위도, 경도` (좌표까지 붙인 확장 형식)
